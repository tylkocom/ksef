# ksef2 SDK 1.0 readiness

This directory tracks the remaining work to move the SDK from the current
`0.x` line into a stable `1.0.0` release.

The previous per-workstream hardening plans were useful while the sync/async,
error-handling, and code-generation refactors were still open. They are now
replaced by the current readiness checklist in
[`01-1.0-readiness.md`](01-1.0-readiness.md).

Use this checklist as the release source of truth:

- keep every item tied to a verification command or file-level evidence;
- mark an item done only after the evidence exists in the current tree;
- add new 1.0 blockers here instead of leaving them in chat history.
